### Wstęp

Omówię tutaj jeden z podstawowych i często sprawiających problemy elementów strony WWW - menu. Wpis ten obejmie tylko kwestie RWD i mechaniki menu, resztę czyli np. zastosowanie glifów lub sprite'ów CSS do nadania menu przejrzystości pozostawiam czytelnikom.

Od kilku lat większość wejść na strony WWW jest z urządzeń mobilnych, dlatego zaczniemy od sprawy podstawowej dla responsive web design (dalej: RWD) czyli tzw. hamburgera. Jest to rozpoznawalna przez użytkowników formuła menu polegająca na umieszczeniu skrótu do menu w typowej postaci trzech równoległych kresek (stąd określenie) umieszczonych w roku ekranu. Dzięki temu menu nie zajmuje miejsca, a po kliknięciu albo rozwija się w dół, albo wysuwa się z boku.

W drugiej części artykułu dodam tzw. Mega Menu (aka Mega-Dropdown Menu), czyli menu składające się z bloków zajmujących najczęściej całą szerokość ekranu, które zawierają wiele odnośników, często również tekst i elementy graficzne.

### Hamburger

Zacznijmy od najprostszego menu, zwykłej belki nawigacyjnej składającej się z listy odnośników w elemencie nav. Będzie to komponent menu.js.

**/src/components/menu.js**

```js
import React from 'react';
import { Link } from "gatsby"

const Menu = () => {
    return (<nav>
        <ul>
            <li><Link to="/">Home</Link></li>
            <li><Link to="/articles">Articles</Link></li>
            <li><Link to="/about">About us</Link></li>
            <li><Link to="/contact">Contact</Link></li>
        </ul>
    </nav>)
}

export default Menu
```

Przykładowy CSS dla tego komponentu wygląda tak:

```css
.menu ul {display: flex; flex-direction: column; padding-left:0; list-style-type: none;}
.menu ul li {flex:1;}
.menu ul li a {display:block; text-align: center; text-decoration: none;}
.menu ul li a:hover {background-color: gold; text-decoration: underline;}

@media (min-width: 800px) {
    .menu ul {flex-direction: row;}
}
```

Jest to podstawowe, typowe stylowanie menu w modelu CSS Flexbox zapewniające układ pionowy w małych rozdzielczościach i poziomy w większych. Zaleta jest prostota oraz to, że zawsze są widoczne wszystkie elementy tego menu. Natomiast wada jest zajmowanie miejsca w pionie co zmusza użytkownika do przewinięcia w dół, co więcej podczas przewijania można niechcący uruchomić element menu i przejść na przypadkową stronę zamiast zobaczyć co jest poniżej.

Żeby uniknąć tych problemów wprowadzono menu hamburger, które zajmuje tylko prostokąt w rogu ekranu. Potrzebny będzie dodatkowy komponent, do którego zaimportujemy powyższe menu. Czyli będzie to wrapper zapewniający odpowiednie zachowanie. Przedstawione tutaj rozwiązanie zaczerpnąłem z artykułu ["Gatsby Navigation using Styled Components and useState hook"](https://snopkowski.com/blog/gatsby-navigation-styled-components) znajdującego się na blogu Wojciecha Snopkowskiego. Trochę go zmieniłem.

Wymogi:

- [Loadable Components](https://loadable-components.com/) zadaniem tego modułu jest code splitting, czyli podział wykonywanych skryptów JS tak, żeby można było załadować menu dopiero po jego utworzeniu, w przeciwnym razie po załadowaniu strony byłby to pusty element i pojawiłby sie dopiero po przeładowaniu.
- [styled-components](https://styled-components.com/) dynamiczny CSS odpowiedzialny za animację hamburgera.

Instalacja obu modułów opisana jest na powyższych stronach, w Gatsby'm przebiega bez zmian. Tylko Styled Components trzeba dodać do `gatsby-config.js`. Szczegóły są opisane we wpisie [Gatsby cz. 3 - CSS](/blog/gatsby-css).

Tak wygląda komponent hamburgera:

**/src/components/hamburger.js**

```js {numberLines}
import React, { useState } from "react"
import styled from "styled-components"
import loadable from "@loadable/component";
const Menu = loadable(() => import("./menu"));

const Navigation = styled.nav`
@media (max-width: 699px) {
    position: absolute;
    height: 2rem;
    top: 0;
    left: 0;
    right: 0;
    left: 0;
           }
`

const Toggle = styled.div`
  display: none;
  height: 100%;
  cursor: pointer;
  padding: 0 10vw;
  background-color: #ddc;

  @media (max-width: 699px) {
    padding-left:1rem;
    display: flex;
  }
`

const Navbox = styled.div`
  display: flex;
  flex-direction: column;
  height: 100%;

  @media (max-width: 699px) {
    position: absolute;
    width: 100%;
    justify-content: flex-start;
    transition: all 0.3s ease-in;
    left: ${props => (props.open ? "-100%" : "0")};
    & h2, h3, ul {font-size:1.3rem; }
    & a:hover {background-color: rgba(251, 251, 251, 0.3); color:#eec;}
  }
`

const Hamburger = styled.div`
  background-color: #ccc;
  width: 30px;
  height: 3px;
  transition: all 0.3s linear;
  align-self: center;
  position: relative;
  transform: ${props => (props.open ? "rotate(-45deg)" : "inherit")};

  ::before,
  ::after {
    width: 30px;
    height: 3px;
    background-color: #ccc;
    content: "";
    position: absolute;
    transition: all 0.3s linear;
  }

  ::before {
    transform: ${props =>
        props.open ? "rotate(-90deg) translate(-10px, 0px)" : "rotate(0deg)"};
    top: -10px;
  }

  ::after {
    opacity: ${props => (props.open ? "0" : "1")};
    transform: ${props => (props.open ? "rotate(90deg) " : "rotate(0deg)")};
    top: 10px;
  }
`

const MenuHamburger1 = () => {
    const [navbarOpen, setNavbarOpen] = useState(false)
    return (
        <Navigation className="menu-hamburger">
            <Toggle onClick={() => setNavbarOpen(!navbarOpen)} >
{navbarOpen ? <Hamburger open /> : <Hamburger />}
            </Toggle>
            {navbarOpen ? (<Navbox onClick={() => setNavbarOpen(false)}> <NavbarLinks /> </Navbox>
            ) : (
<Navbox open> <NavbarLinks />  </Navbox>)}
        </Navigation>
    )
}
export default MenuHamburger1
```

Po kolei:

- Na samym początku pliku mamy import wszystkich niezbędnych modułów: React z hookiem useState, Styled Components, Loadable Components i wreszcie przez Loadable Components importujemy właściwe menu. Dzięki temu będziemy je mieli od razu po pierwszym załadowaniu strony.
- Pomińmy cały CSS w Styled Components i przejdźmy do samego komponentu: najpierw utworzony jest stan navbarOpen ustawiony na false i metoda zmieniającą setNavbarOpen; potem zwracany jest element hamburgera zawarty w znaczniku nav umieszczonym tu jako komponent Styled Components, jest to wrapper zawierający dwa składniki: przełącznik stanu (komponent Styled Components Toggle, czyli właśnie sam hamburger) oraz operator warunkowy (ternary operator), który zależnie od stanu tworzy element Navbox.
- Toggle jest przełącznikiem, jest widoczny tylko w małej rozdzielczości i zmienia stan oraz zawiera operator warunkowy, który zależnie od stanu zmienia wygląd hamburgera (element Hamburger). Każde kliknięcie zmienia stan na przeciwny, a wizualnie oddane jest to trzema równoległymi kreskami kiedy hamburger jest zamknięty i iksem kiedy jest otwarty. Jeżeli stan jest true Hamburger posiada propsa open, jeżeli stan jest false nie ma go. Zaraz zobaczymy jakie to ma znaczenie.
- operator warunkowy zarządza zasadniczą częścią menu, jest to div (Navbox) zawierający menu, który mając stan false znajduje się poza ekranem, a kiedy stan zmienia się na open wyjeżdża z lewej krawędzi ekranu. Kliknięcie zmienia stan na false, co zapewnia płynne działanie w Landing Page oraz w sytuacji jeśli klikniemy na link podstrony aktywnej. A jak logiczny stan komponentu zmienia jego wygląd? Dokładnie tak samo jak w opisanym powyżej elemencie Hamburger. Jeżeli stan jest false nie ma propsa open, a jeżeli stan jest true jest props open.
- Wróćmy do CSS Styled Components, tutaj w elementach Hamburger i Navbox widzimy propsy użyte jako stany logiczne dla operatorów warunkowych sterujących wyglądem tych elementów. Elementy wizualne Hamburgera, czyli te trzy kreski to div z dwoma pseudoelementami CSS (kreski dolna i górna), których płynność animacji zapewnia 'transition: all 0.3s linear;'. Podobnie wygląda to w przypadku Navboxa, tutaj również mamy transition z tym samym czasem i pozycja left która zależnie od propsa wynosi -100% lub 0.

### Dropdown Menu

Najbardziej typowym rozwiązaniem dla rozwijanego menu jest umieszczenie listy wewnątrz elementu listy nadrzędnej z właściwością `display: hidden;`. W wyniku akcji na elemencie nadrzędnym (kliknięcie lub najechanie) następuje zmiana na `display:block`.

Wygląda to tak:

**HTML**

```html
<nav class="menu-dropdown">
        <ul>
            <li><a href="/">Home</a></li>
            <li><a href="/articles">Articles</a>
<ul>
<li><a href="/article1">Article 01</a></li>
<li><a href="/article2">Article 01</a></li>
<li><a href="/article3">Article 01</a></li>
<li><a href="/article4">Article 01</a></li>
</ul></li>
            <li><a href="/about">About us</a></li>
            <li><a href="/contact">Contact</a></li>
        </ul >
    </nav >
```

**CSS**

```css {numberLines}
a:hover {background-color: #ddc;}
ul {padding-left:0; list-style-type: none;}
.menu-dropdown > ul {display: flex; flex-direction: column;}
.menu-dropdown ul li {flex: 1; position: relative;}
.menu-dropdown ul li ul {display: none; position: absolute; width:100%;}
.menu-dropdown ul li:hover ul {display: block;}
.menu-dropdown ul a {display: block; text-align: center;}

@media (max-width: 800px) {
  .menu-dropdown ul li:hover ul {display:none;}
}

@media (min-width: 800px) {
    .menu-dropdown ul {flex-direction: row;}
}
```

Powyższe reguły zapewniają:
- niewyświetlenie listy podrzędnej `display:none` i wyświetlenie jej w wyniku najechania myszką `display:block`
- wyświetlona w ten sposób lista umieści się absolutnie `position: absolute;`, ale względem elementu nadrzędnego `position: relative;` i będzie miała jego całkowitą szerokość `width:100%;`.
- dodatkowo w media query trzeba zadbać, żeby lista podrzędna nie wyświetlała się w ogóle w małych rozdzielczościach.

To jest tylko szkieletowe rozwiązanie HTML+CSS, które łatwo można dostosować do przedstawionego w pierwszym rozdziale Hamburgera i ostylować.

### Mega Menu

Specyficznym rodzajem rozwijanego menu jest tzw. Mega Menu, czyli Dropdown, którego blok zajmuje całą szerokość bloku treści.

Żeby dla powyższego przykładu uzyskać dokładnie taki efekt wystarczy wprowadzić dwie zmiany w CSS:

- przenieść `position:relative;` o poziom wyżej, czyli z elementu listy nadrzędnej na samą listę nadrzędną
- dla wyświetlonej listy podrzędnej dodać właściwość `left: 0;`

Na tym etapie uzyskaliśmy efekt Mega Menu, ale do prawdziwego Mega jeszcze wiele brakuje.

Po pierwsze zasadniczą ideą Mega Menu jest umieszczenie wielu elementów w bloku Dropdown, do tego często są one dość zróżnicowane, nie tylko listy ale i buttony, ilustracje, bloki tekstu a nawet formularze. Będziemy potrzebowali dodatkowego elementu HTML, który będzie kontenerem dla tych elementów.

Po drugie wyświetlenie dużego bloku po najechaniu na element menu nie powinno następować natychmiast. Dobrze jest dodać element animacji, opóźnienia akcji. To niestety nie jest trywialnie proste, bo dla zmiany właściwości `display` nie możemy zastosować `transition: all .5s;` to nie zadziała.

Zamieńmy więc listę podrzędną z przykładu z poprzedniego rozdziału na poniższy blok treści:

```html
<div>
    <div>
        <h5>Section 1</h5>
        <ul>
            <li><Link to="/article101">Article 101</Link></li>
            <li><Link to="/article102">Article 102</Link></li>
            <li><Link to="/article103">Article 103</Link></li>
            <li><Link to="/article104">Article 104</Link></li>
        </ul>
        <button>Click me</button>
    </div>
    <div>
        <h5>Section 2</h5>
        <ul>
            <li><Link to="/article202">Article 202</Link></li>
            <li><Link to="/article203">Article 203</Link></li>
            <li><Link to="/article204">Article 204</Link></li>
            <li><Link to="/article201">Article 201</Link></li>
        </ul>
        <button>Click me</button>
    </div>
    <div>
        <h5>Section 3</h5>
        <ul>
            <li><Link to="/article301">Article 301</Link></li>
            <li><Link to="/article302">Article 302</Link></li>
            <li><Link to="/article303">Article 303</Link></li>
            <li><Link to="/article304">Article 304</Link></li>
        </ul>
        <button>Click me</button>
    </div>
</div>
```

CSS z niewielkim zmianami:

```css {numberLines}
a:hover {background-color: #ddc;}
ul {padding-left:0; list-style-type: none;}
.menu-dropdown > ul {display: flex; flex-direction: column; position: relative;}
.menu-dropdown ul li {flex: 1;}
.menu-dropdown ul li > div {display: none; position: absolute; left:0; width:100%;}
.menu-dropdown ul li:hover > div {display: flex;}
.menu-dropdown ul li:hover > div div {flex:1;}
.menu-dropdown ul a {display: block; text-align: center;}
@media (max-width: 800px) {
  .menu-dropdown ul li:hover div {display:none;}
}

@media (min-width: 800px) {
    .menu-dropdown > ul {flex-direction: row;}
}
```

Pozostają do rozwiązania dwie kwestie:

- animacja, opóźnienie wyświetlania bloku
- zintegrowanie Mega Menu z Hamburgerem, na tym etapie cała treść elementu Mega Menu po prostu znika i widać tylko odnośnik menu nadrzędnego; jeżeli prowadzi od do podstrony zawierającej tę samą treść problem jest rozwiązany, jeśli jest inaczej mamy poważny problem z dostępnością i w ogóle w sensie nawigacji ten element jest bezużyteczny

Wbrew pozorom - jeżeli chcemy zastosować opóźnienie, od tego trzeba zacząć. Zmiany, które wprowadzimy mogą mieć wpływ na integrację.

Ponieważ właściwość `display` nie może być animowana, trzeba docelowy `display:flex;` przenieść do właściwości CSS elementu div przed najechaniem i manipulować elementem przy pomocy zmieniającej się właściwości `opacity`. Do tego trzeba dodać `left:-100%;` żeby uniknąć efektu stopniowego znikania. Natomiast sama zmiana opacity będzie spowolniona przez `transition: opacity .5s ease-in-out;`,

Docelowo wygląda to tak:

```css {numberLines}
a:hover {background-color: #ddc; transition: .5s;}
ul {padding-left:0; list-style-type: none;}
.menu-dropdown > ul {display: flex; flex-direction: column; position: relative;}
.menu-dropdown ul li {flex: 1;}
.menu-dropdown ul li:hover > a {background-color: #ddc;}
.menu-dropdown ul li > div {display: flex; position: absolute; opacity:0; left:-100%; transition: opacity .5s  ease-in-out; background-color: #fff; box-shadow: 0 2px 5px rgba(51,51,51,.3); padding: .5rem;}
.menu-dropdown ul li:hover > div { max-width:100%; height: auto; opacity:1; left:0;}
.menu-dropdown ul li:hover > div div {flex:1;}
.menu-dropdown ul a {display: block; text-align: center;}

@media (max-width: 800px) {
  .menu-dropdown ul li:hover div {display:none;}
}

@media (min-width: 800px) {
    .menu-dropdown > ul {flex-direction: row;}
}
```

### Mega Menu + Hamburger

Na samym początku trzeba się zastanowić jak ma wyglądać mechanika menu w formie Hamburgera. Na ten moment całą treść bloku Mega Menu mamy ukrytą. Można utworzyć osobną podstronę i tam przekierowywać użytkownika, ale jest to po pierwsze mało wydajne, a po drugie mylące dla użytkownika.

Rozwiązaniem jest submenu (menu drugiego rzędu).

Najprościej zrobić to tak jak jest w menu [Adobe](https://www.adobe.com/), po kliknięciu submenu rozwija się przesuwając resztę menu w dół. To jest dobre rozwiązanie jeśli submenu nie ma wielu elementów.

Bardziej ambitne i zaawansowane jest takie, które nasuwa się na rozwiniętego Hamburgera i tak samo się zachowuje, żeby je schować trzeba kliknąć iksa: jest to Hamburger w Hamburgerze. Przykładem może być menu zastosowane na stronie [Atlassian](https://www.atlassian.com/).

#### Hamburger + Dropdown

Zacznijmy od pierwszego rozwiązania. Najprościej jest zrobić to samym CSS-em i takim przykładowym kodem jest zmiana poprzedniej wersji w tym fragmencie:

```css {numberLines}
@media (max-width: 700px) {
  .menu-dropdown ul li div {display:none; opacity:0; transition: opacity .5s ease-in-out;}
  .menu-dropdown ul li:hover div {display: flex; flex-direction: column; position:relative; opacity:1;}
}
```

Ma to dwie wady:

- zmiana następuje błyskawicznie i może to być mylące
- nie ma oczywistego wyjścia z wyświetlenia submenu

Trzeba to zrobić na kliknięcie. Pierwsze powoduje wysunięcie podmenu, drugie jego zamknięcie. W tym celu umieszczamy w odnośniku menu nadrzędnego wywołanie funkcji która dodaje i usuwa klasę CSS z następnego elementu HTML:

```js {numberLines}
//
let flag = true;
const Menu = () => {

    const handleSubmenu = (e) => {
        const divSubmenu = e.target.nextElementSibling;
        if (flag) { divSubmenu.classList.add("expanded"); flag = false; console.log(divSubmenu.classList) }
        else { divSubmenu.classList.remove("expanded"); flag = true; }
    }

    return (<nav className="menu-dropdown">
        <ul>
            <li><Link to="/">Home</Link></li>
            <li><Link to="#" onClick={(e) => handleSubmenu(e)}>Articles</Link>
                <div>
                  //
```
I analogiczna zmiana tego samego fragmentu kodu CSS:

```css {numberLines}
@media (max-width: 700px) {
  .menu-dropdown ul li > div {display:none; }
  .menu-dropdown ul li > div.expanded {display: flex; flex-direction: column; position:relative; left:0; opacity:1;}
}

@media (min-width: 700px) {
    .menu-dropdown > ul {flex-direction: row;}
    .menu-dropdown ul li:hover > div { width:100%; max-width:99vw;}
}
```
